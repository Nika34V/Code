## Анализ результатов

### Ключевые выводы

1. **Максимальное ускорение:** 4.85x (400×400, 8 потоков)

2. **Оптимальная конфигурация:**
   - Для малых матриц (200-400): 4-8 потоков
   - Для больших матриц (≥800): 8 потоков

3. **Эффективность:**
   - 2 потока: 94-102% (почти линейное ускорение)
   - 4 потока: 60-99% (хорошая масштабируемость)
   - 8 потоков: 43-60% (ограничено архитектурой)

4. **Производительность:**
   - Пик: 2.67 GFLOPS (400×400, 8 потоков)
   - Снижение для больших матриц из-за кэш-промахов

### Проблемы и наблюдения

1. **Аномалия для 800×800 с 4 потоками:**
   - Ускорение 2.41x (эффективность 60%)
   - Возможная причина: конфликт кэшей между потоками

2. **Снижение производительности на 2000×2000:**
   - Данные не помещаются в кэш
   - Увеличение задержек памяти
