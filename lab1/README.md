## 5. Выводы

### Корректность работы
Программа на C++ успешно перемножает квадратные матрицы любых размеров. Результат полностью совпадает с эталоном NumPy для всех проверенных размеров (5, 200, 400, 800, 1200, 1600, 2000). Это подтверждает правильность реализации вычислительного алгоритма.

### Соответствие теоретической сложности
Временные затраты растут пропорционально N³, что соответствует теоретической оценке O(N³). Количество операций (2N³) также совпадает с расчётным.

### Производительность
- На малых размерах (N≤200) производительность высокая (до 830 MFLOPS) благодаря эффективному использованию кэша.
- С ростом N производительность падает, особенно при N > 800, когда матрицы перестают помещаться в кэш-память процессора.
- Для N=2000 производительность составляет всего ~105 MFLOPS – это ожидаемо для наивной реализации без оптимизаций (блочное умножение, векторизация, многопоточность).
