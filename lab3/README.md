## Анализ результатов

### Ключевые выводы

1. **Время выполнения** растет пропорционально O(N³):
   - 200×200: 37.7 мс
   - 2000×2000: 82616.6 мс (в 2190 раз больше)

2. **Производительность:**
   - Стабильна для матриц до 1600×1600: ~0.4 GFLOPS
   - Падает до 0.19 GFLOPS для 2000×2000

3. **Ускорение:**
   - Для 1 процесса ускорение ~1x (ожидаемо)
   - Небольшие отклонения из-за погрешности измерений

### Проблемы и наблюдения

1. **Эффективность >100% для 200×200:**
   - Связано с погрешностью измерений малых времен

2. **Снижение производительности на 2000×2000:**
   - Данные не помещаются в кэш-память
   - Увеличение задержек при обращении к RAM
